# Spot the AI

FaceMash-style game: two short passages are shown, one written by a human author, one generated by Claude. Pick the AI.

Live at https://spot-the-ai.oiatc.ca.

## Run locally

```bash
npm install
npm start            # or: npm run dev
```

Open http://localhost:3000.

## Architecture

The game serves from a **pre-generated pool** (`pool.json`) that is committed to the repo and baked into the production image. The server makes **no Claude API calls at runtime** — it just picks a random round, randomizes which side is AI, and scores the guess. This keeps the public endpoint cheap and abuse-proof.

- `seed.json` — public-domain human passages from Project Gutenberg, each tagged with a `topic` describing voice/era.
- `generate-pool.js` — offline generator. For each seed, asks Claude for several original passages matching the seed's voice, era, word count, and sentence count (length is the main "tell", so it's matched tightly). Writes `pool.json`.
- `server.js` — Express server. `GET /api/round` returns a random pair (sides randomized); `POST /api/guess` scores against in-memory truth. `GET /healthz` for the container healthcheck.
- `public/index.html` — Vue 3 (CDN) single-page UI: two cards, score tracker, reveal-on-guess with source attribution, and a Wordle-style Share button.

## Regenerating the pool

Requires an Anthropic API key:

```bash
ANTHROPIC_API_KEY=sk-ant-... npm run generate -- 5   # 5 variants per seed
```

Commit the updated `pool.json` and redeploy (bump `SPOT_THE_AI_REF` in waaseyaa-infra).

## Deployment

Containerized and served under the oiatc.ca Cloudflare Tunnel + Caddy stack. The Dockerfile and compose wiring live in `waaseyaa-infra` (`compose/spot-the-ai/`); CI rebuilds the container on push. See that repo's `runbooks/03-add-a-site.md`.
